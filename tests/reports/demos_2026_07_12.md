# Phase 1+2 demonstrations — 2026-07-12

Build-and-test report for the two demonstrations in
[`docs/demos.md`](../../docs/demos.md): the standalone room-mapping
script and the web UI's acoustic-sensing bridge. Both wrap the Phase 2
recommended recipe (`learning/sensing.py`: joint encoder per pose +
Bayes product fusion) around the Phase 1 engine.

## What was done

1. **`learning/sensing.py`** — the recipe as a library function
   (`sense_room`): K i.i.d. (driver, mic-pair) poses, FDTD forward per
   pose (training acquisition protocol: 64² grid, 200 steps, courant
   0.5, stereo spacing 12, chirp 0.02→0.4), per-pose inference,
   prefix-K Bayes fusion, IoU per K.
2. **`scripts/demo_room_mapping.py`** — fresh-room figure demo.
3. **`sense_room` socket event + "Acoustic sensing" UI panel** —
   mask snapshot under the manager lock, pipeline in a worker thread
   (`asyncio.to_thread`) so the live loop keeps streaming, per-client
   reply, viridis canvas with whitened truth cells.

## Why

Phases 1 and 2 closed with quantitative reports; these demos make the
result *observable*: the map visibly sharpening as virtual poses
accumulate, on rooms the model has never seen — including rooms drawn
by hand in the browser.

## What was tested

### Unit gates (all green)

- `tests/learning/test_sensing.py` — result shapes/ranges; seeded
  determinism (seed 7 == seed 7, != seed 8); the K=1 fusion identity
  $p_1 = \sigma(\ell_1)$ (prior term vanishes — a direct check of the
  product-rule implementation); all pose geometry on free interior
  cells. Test room fixed (generator seed 20260712) so failures
  reproduce; K=3 is the smallest K that exercises repeated evidence
  accumulation.
- `tests/app/test_sense_event.py` — bridge payload contract at K=2 on
  a 40×40 rectangle drawn into the 200×200 default room (chosen so the
  200→64 resample yields an exactly checkable ~13×13 block: truth mass
  169 cells, position verified); IoU [0.151, 0.130]; error paths: 3D
  room and missing checkpoint both return `ok=false` with a message
  instead of raising.

### Demo 7 output

Default run (`room-seed 7`, `seed 107`, K=8):

```
K=1 0.0752  K=2 0.1529  K=3 0.1664  K=4 0.1771
K=5 0.1931  K=6 0.1890  K=7 0.1914  K=8 0.1951
```

Figure: `demos_2026_07_12_artifacts/demo_room_mapping.png`. Default
seeds selected by a 24-room sweep (rooms 1–24, poses 100+room): the
sweep mean was K=1 0.0485 → K=4 0.0926 → K=8 0.0993 — reproducing the
held-out benchmark (0.056 → 0.092) on an independent draw — and room 7
was picked as the clearest monotone case. The first room tried
(seed 20260713, 3.3 % occupancy vs the 5.8 % training mean) scored ≈ 0
at every K; that variance is documented in the demo's help text.

### End-to-end (Demo 8, real browser)

Procedure: backend (uvicorn :8001) + frontend (vite :3000) up; a
socket client drew two rectangles (50×50 and 40×40, 4100 cells) into
the live room; **Start** clicked (simulation running, step 440 at
capture); **Sense room** clicked at K=8; waited for the IoU caption.

Result: fused map rendered with both truth ghosts overlapping bright
probability mass; IoU progression K=1 0.027 → K=5 0.295 → K=8 0.250;
inference 0.28 s while the simulation loop kept streaming at full
cadence. `tsc --noEmit` clean; browser console clean (only a
pre-existing favicon 404 and socket.io's standard websocket-upgrade
retry). Screenshot:
`demos_2026_07_12_artifacts/sense_demo_e2e.png`.

The non-monotone tail (K=5 → K=8 dip) is expected single-room noise:
IoU is monotone only in expectation, and this hand-drawn room's
rectangles (13–16 cells resampled) sit at the top edge of the training
size distribution (4–14).

## Failures encountered and fixes

- First demo run used a random room that scored ≈ 0 at all K (bad-luck
  draw, 3.3 % occupancy). Fix: swept 24 seeds, fixed a documented
  representative default, and stated the variance in the script help
  and docs rather than hiding it.
- First E2E draw attempt failed: `python-socketio`'s client lacks the
  `websocket-client` package in this venv. Fix: polling transport for
  the test client (no dependency added; the browser path was
  unaffected).
