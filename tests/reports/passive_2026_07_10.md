# Passive-sensing baseline — 2026-07-10

Task 2.2.2: train the blind-deconvolution model (`PassiveCNN`, source
signal unknown) under conditions matched to the active baseline, and
compare. Companion to [training_2026_05_14.md](training_2026_05_14.md)
(active baseline) and [multipose_2026_07_10.md](multipose_2026_07_10.md).

## What

`PassiveCNN` (198,529 params — `DualInputCNN` with the source branch
removed, all shared hyperparameters identical) trained on 10 000
samples with **per-sample randomised chirps**
(`--randomize-source`: $f_0 \sim U[0.01, 0.10]$,
$f_1 \sim U[0.20, 0.45]$; seed 5678), evaluated on the 10k in-dist
archive and a 500-sample held-out archive (seed 8765). Optimiser,
schedule, epochs, batch size, split seed all match the active baseline
(AdamW lr 1e-3 / wd 1e-4, cosine, 100 epochs, batch 32, seed 42).

## Why

Task 2.2 asks how much of the active model's (already weak) skill
depends on knowing the source. Randomising the source is what makes
the passive setting honest — with the fixed default chirp, a
"passive" model could memorise the constant excitation and the
comparison would be meaningless. The source randomisation also makes
the *task* harder (echo timing must be read without a reference), so
the passive-vs-active gap bounds the value of source knowledge from
above.

## Results

| model | source | in-dist IoU | held-out IoU | best val IoU |
| --- | --- | --- | --- | --- |
| active (`DualInputCNN`) | known, fixed chirp | 0.134 | 0.037 | 0.042 |
| passive (`PassiveCNN`) | unknown, randomised chirp | 0.099 | 0.030 | 0.037 |

Training curves (`checkpoints/passive_baseline/loss.png`, epochs on x;
loss / IoU on y): the same signature as every single-pose run — val
loss rises monotonically from ~1.11 while train IoU climbs to 0.227;
the model memorises from the first epoch. Prediction grids are in
`passive_2026_07_10_artifacts/` (`preds_indist.png`,
`preds_heldout.png`): held-out predictions again cluster in
prior-favoured regions rather than tracking per-scene obstacles.

## Interpretation

1. **The passive model retains most of the active model's (limited)
   held-out skill** (0.030 vs 0.037, −19 %) despite a harder task.
   This is consistent with the under-determination diagnosis: the
   active model was never successfully *using* the source reference
   for inverse mapping, so removing it costs little. The gap between
   the settings should widen once the sensing side is no longer the
   bottleneck (i.e. with multi-pose data).
2. **In-dist memorisation is weaker** (0.099 vs 0.134): randomised
   sources make waveform fingerprints less memorisable — the same
   effect direction the augmentation run showed, here induced by task
   design rather than noise injection.
3. Both numbers remain far below the multi-pose Bayes fusion (0.092
   held-out at K=8): pose diversity is worth more than source
   knowledge at this grid size. Passive multi-pose is the natural
   follow-up after 2.1.4c.

Known limitation (documented in `docs/learning.md` §7): the power-
spectrogram front-end discards phase and therefore inter-channel time
differences — the strongest passive cue. A GCC-PHAT cross-correlation
input channel is the designated upgrade if passive performance
matters downstream.

## Test-data rationale

Sample counts, grid, room statistics, and seeds mirror the active
baseline exactly, so the only changed variables are the model's input
(no source branch) and the source distribution (randomised). Seeds
5678/8765 are disjoint from all active-sensing streams (1234, 999,
424242, 31415).
